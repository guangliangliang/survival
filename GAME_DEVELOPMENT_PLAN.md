# 《守卫村庄：幸存者》开发计划

## 总体方向

- 微信横屏俯视角幸存者游戏，不加入塔防和建造。
- 玩家扮演村庄守卫，持枪在村庄外围大地图战斗。
- 武器自动索敌、自动开火，玩家负责移动、走位和升级选择。
- 村庄作为局外任务基地和世界观入口，不设置战斗中的村庄血条。
- 第一版采用固定手工地图、12 分钟限时生存和局内升级。

## 1. 策划阶段

首个关卡为“清理村庄外围”，地图包含村口、农田、森林、河边和废弃强盗营地。

- 0–4 分钟：野狼、野猪等动物。
- 4–8 分钟：动物与近战强盗混合出现。
- 8–11 分钟：增加远程强盗、精英敌人与更高密度兽潮。
- 11–12 分钟：生成强盗头目；坚持 12 分钟并击败头目后胜利。
- 玩家生命归零立即失败。

首版内容控制为 1 名角色、1 把初始步枪、4 种普通敌人、1 种精英敌人、1 个首领。敌人死亡掉落经验物，升级时暂停并随机出现三个选项。升级池包括伤害、射速、穿透、弹丸数量、射程、移动速度和最大生命。

## 2. 代码阶段

1. 修复死亡结算、暂停、重新开始和场景状态清理。
2. 建立统一局内状态，完成经验掉落、拾取和升级三选一。
3. 使用 Godot Resource 数据化武器、敌人和升级配置。
4. 制作固定大地图，按地图区域和时间阶段生成敌人。
5. 完成 12 分钟首领战与胜利结算。
6. 添加微信横屏虚拟摇杆、响应式 HUD 和安全区域布局。
7. 为高频对象实现对象池、数量上限、镜头外回收和移动端性能保护。

关键运行时接口：

- `GameManager`：局内时间、等级、经验、击杀和胜负状态。
- `WeaponData`：伤害、射速、射程、弹速、穿透和弹丸数量。
- `EnemyData`：生命、速度、伤害、经验、生成阶段和区域。
- `UpgradeData`：升级名称、等级上限、属性类型和修改量。
- `InputAdapter`：统一读取键盘与虚拟摇杆输入。

## 3. 美术阶段

代码阶段使用色块和简单图标。正式资源默认采用俯视角乡村像素风，角色建议 64×64 像素，地图使用 32×32 或 64×64 瓦片。资源优先级为角色与敌人辨识度、攻击反馈、地图、HUD、菜单。

## 4. 音乐与音效阶段

先制作开枪、命中、死亡、受伤、经验拾取、升级、首领出现和按钮点击等反馈音效，再制作村庄菜单、普通战斗和首领战音乐。使用 OGG 格式，并提前建立音乐、音效和 UI 音频总线接口。

## 测试与验收

- Godot 4.5.1 无脚本解析错误，主菜单和战斗场景均能启动。
- 键盘和触屏摇杆均能完成一局游戏。
- 升级、暂停、死亡、胜利、重开和返回菜单流程正确。
- 首领仅在最终阶段出现，达到 12 分钟且首领死亡后胜利。
- 高频对象能够回收，无持续内存增长。
- 微信真机以中端设备为基准，压力目标约为 120 个活跃敌人和 200 个活跃投射物。

## 首版边界

首版只制作一张地图和一个角色，不制作塔防、自由建造、联机、永久数值成长、装备制造和完整村庄 NPC 系统。
