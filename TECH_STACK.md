# 技术选型

## 一、核心引擎与语言

| 技术 | 版本 | 用途 |
|------|------|------|
| **Godot Engine** | 4.5.1 | 游戏引擎，2D渲染、物理、场景管理 |
| **GDScript** | Godot 4.x 语法 | 主要编程语言，性能优秀且语法简洁 |

### 为什么选择 GDScript？
- 更轻量，包体更小
- 原生支持 Godot 所有功能
- 学习曲线平缓，开发效率高
- 更适合微信小游戏平台

## 二、渲染与物理
- **渲染器**：Godot 2D 渲染引擎（Forward+）
- **物理引擎**：Godot Physics 2D
- **碰撞检测**：Built-in 2D 碰撞系统

## 三、微信小游戏平台

| 技术 | 说明 |
|------|------|
| **Godot 微信小游戏导出模板** | Godot 官方提供的微信小游戏导出支持 |
| **微信小游戏 API** | 登录、分享、排行榜、广告等 |
| **微信开发者工具** | 调试、预览、上传代码 |

## 四、开发工具链

| 工具 | 用途 |
|------|------|
| **VS Code** | 代码编辑器（配合 Godot GDScript 插件） |
| **Git** | 版本控制 |
| **Aseprite/Photoshop** | 像素画/美术资源制作 |
| **Audacity** | 音频编辑 |
| **TexturePacker** | 图集打包（可选） |

## 五、资源格式
- **图片**：PNG / WebP（推荐 WebP 以减小体积）
- **音频**：OGG Vorbis
- **字体**：TTF / OTF
- **动画**：Sprite Sheet（精灵表）

## 六、架构模式
- **ECS架构**：使用Godot的节点系统实现类似ECS的架构
- **单例管理器模式**：通过Autoload实现全局管理器
- **状态机模式**：玩家、敌人AI使用状态机
- **事件驱动**：使用Godot信号系统实现模块解耦

## 七、微信小游戏优化要点

### 7.1 资源优化
- 图片压缩（WebP格式）
- 音频压缩（OGG）
- 图集打包

### 7.2 代码优化
- 对象池（子弹、敌人）
- 减少物理计算
- 视距剔除

### 7.3 包体优化
- 初始包体控制在4MB以内
- 资源分包加载
